CSS3

CSS is a style sheet language used for describing the presentation of a document written in a markup language. It can be applied not just in HTML and XHTML, but in XML, SVG and XUL and is applicable to rendering in speech or media.

It was designed primarily to enable the separation of document content from document presentation, including aspects such as the layout, colors and fonts.

It is a prototype-based with first-class functions supporting object-oriented, imperative and functional programming styles.

RESPONSIVE VS ADAPTIVE DESIGN

RESPONSIVE DESIGN

Responsive design respond to the size of the browser at any given point with fluid layouts. It doesn't matter what size is the browser, the layout will adapt their content.

In reponsive design we focus on our layout design and when it needs to be readapted.

  • Media queries
  • Relative layout size units: %, vh, vw, em, rem
  • Relative font size units: em, rem

ADAPTIVE DESIGN

Adaptive design uses static layouts based on breakpoints which don't respond once they are initially loaded.

The adaptive design detect the browser size and load the appropiate layout for it. In this case, we have to take into consideration the different resolutions of the target devices. The most common widths are 320, 480, 768, 960, 1024, 1200 and 1600.

  • Media queries
  • Absolute layout size units: px
  • Relative or asbolute font size units: px, em, rem

MOBILE FIRST VS DESKTOP FIRST

MOBILE FIRST

Mobile first is a progressive enhancement strategy which takes a bottom-up approach where you first design for the mobile device and then embellish for the bigger devices.

PREPROCESSORS

A preprocessor is a program that takes an input data to produce output that is used as input to another program. We mainly use preprocessor in the case of HTML and CSS to facilitate the daily work. Thanks to preprocessor we gave some benefits to languages that originally they don't have, in the case of CSS, we can work with inheritance, variables or conditional flows for example.

The benefits of using a preprocessor are:

  • We are not adding more lines of code to our website, we are just using an improve language to generate the final css that we are going to use in our website.

  • Easy to maintain

  • DRY (Don't repeat yourself)

  • CSS more organized

MICROSOFT .NET

Microsoft .NET is a programming framework created by Microsoft to create applications more easily. Microsoft offers a fast, economy, robust and secure way of develop software/solutions allowing a fast and agile integration between companies.

Languages: C#, Visual Basic, XAML, C++ or J# among others.

Class Libraries: ASP.NET (Web components), ADO.Net (for data), Windows Forms (Desktop applications), WCF (service-oriented applications), WPF (Modern desktop applications), etc.

NODE JS

Node.js is an open-source, cross-platform JavaScript runtime environment for developing a diverse variety of server tools and applications. Although Node.js is not a JavaScript framework, many of its basic modules are written in JavaScript, and developers can write new modules in JavaScript. The runtime environment interprets JavaScript using Google's V8 JavaScript engine.

Node.js has an event-driven architecture capable of asynchronous I/O. These design choices aim to optimize throughput and scalability in Web applications with many input/output operations, as well as for real-time Web applications (e.g., real-time communication programs and browser games)

ASP.NET MVC

ASP.NET MVC is a web application framework developed by Microsoft which implements the model-view-controller pattern.

It allows software developers to build as a composition of three roles: Model, View and Controller.

The view engines used in the ASP.NET MVC 3 and MVC 4 frameworks are Razor and the Web Forms. Both view engines are part of the MVC 3 framework. By default, the view engine in the MVC framework uses Razor .cshtml and .vbhtml, or Web Forms .aspx pages to design the layout of the user interface pages onto which the data is composed

  • Model: Represents the state of a particular aspect of the application. and then passes information to the view.

  • View: Accepts necessary information from the controller and renders a user interface to display that information

  • Controller: Handles interactions from the view and updates the model to reflect a change in state of the application.

REST AND SOAP

REST API

REST web services are a architectural style provide interoperability between computer systems on the internet. Allows requesting systems to access and manipulate textual representations of Web resources using a uniform and predefined set of stateless operations.

  • Client-Server Communication: Client-server architectures have a very distinct separation of concerns.

  • Stateless: The server must be able to completely understand the client request without using any server context or server session state.

  • Cacheable: Cache constraints may be used, thus enabling response data to be marked as cacheable or not-cacheable. Any data marked as cacheable may be reused as the response to the same subsequent request.

SOAP

SOAP is a protocol specification for exchanging structured information in the implementation of web services in computer networks. It uses XML Information Set for its message format, and relies on application layer protocols, most often Hypertext Transfer Protocol (HTTP) or Simple Mail Transfer Protocol (SMTP), for message negotiation and transmission.

SOAP provides the Messaging Protocol layer of a web services protocol stack for web services. It is an XML-based protocol consisting of three parts:

  • an envelope, which defines the message structure and how to process it

  • a set of encoding rules for expressing instances of application-defined datatypes

  • a convention for representing procedure calls and responses

DESKTOP APPLICATIONS

WINDOWS PRESENTATION FOUNDATION

Windows Presentation Foundation (or WPF) is a graphical subsystem by Microsoft for rendering user interfaces in Windows-based applications. Rather than relying on the older GDI subsystem, WPF uses DirectX.

WPF attempts to provide a consistent programming model for building applications and separates the user interface from business logic. It resembles similar XML-oriented object models, such as those implemented in XUL and SVG.

ELECTRON

Electron is an open-source framework developed by GitHub. It allows for the development of desktop GUI applications using the Node.js runtime and the Chromium web browser, originally used for the development of backend web applications.

A basic Electron app consists of three files: package.json (metadata), main.js (code) and index.html (graphical user interface). The framework is provided by the Electron executable file (electron.exe in Windows, electron.app on macOS and electron on Linux).

APP NATIVE VS APP HIBRID

APP NATIVE

Native apps are designed and coded for a specific kind of device. iPhone apps are written in Swift or Objective-C and Android apps in Java.

Ventages of writing native apps:

  • Great performance

  • Better UX: both Android and iOS apps have some individual characteristics, which feel natural for their users.

  • Easier access to built-in capabilities: native apps have no problem tapping into all the device‚Äôs functionalities, such as: camera, microphone, calendar, GPS etc. while hybrid apps grant only a limited access to them.

  • Internet connection is not always a must

APP HIBRID

Hybrid apps are basically web apps hidden behind a native app shell. They are cross-platform and can be immediately distributed between app stores without the need to develop two different versions for Android and iOS.

Most of hybrid apps are built using cross-compatible web technologies (HTML5, CSS, Javascript etc.) and then they are wrapped in a native app using platforms such as Cordova.

Ventages of writing hibrid apps:

  • Cost: the lowered cost of developing a hybrid app instead of a native application is one of the biggest pros of this solution.

  • Lower requirements: one language can be used on all available platforms (assuming you are using Cordova or PhoneGap wrapper)

DATABASES

SQL

SQL is a domain-specific language used in programming and designed for managing data held in a relational database management system (RDBMS), or for stream processing in a relational data stream management system (RDSMS).

NOSQL

NOSQL provides a mechanism for storage and retrieval of data which is modeled in means other than the tabular relations used in relational databases.

The data structures used by NoSQL databases (e.g. key-value, wide column, graph, or document) are different from those used by default in relational databases, making some operations faster in NoSQL.

VERSION CONTROL

A component of software configuration management, version control, also known as revision control or source control, is the management of changes to documents, computer programs, large web sites, and other collections of information.

Changes are usually identified by a number or letter code, termed the "revision number", "revision level", or simply "revision". For example, an initial set of files is "revision 1". When the first change is made, the resulting set is "revision 2", and so on. Each revision is associated with a timestamp and the person making the change. Revisions can be compared, restored, and with some types of files, merged.
